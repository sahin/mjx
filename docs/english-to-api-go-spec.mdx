# English-to-API Engine (Go): Architecture & Specification

**Author:** Manus AI
**Date:** March 10, 2026

## 1. Introduction

This document outlines the architecture for an **English-to-API Engine**, a compiler that translates natural language descriptions of an API's resources and endpoints into a complete, production-ready backend service written in Go. The engine is designed to automate the creation of RESTful APIs, generating idiomatic Go code for data models, HTTP handlers, and server scaffolding. This dramatically reduces the time and effort required to build robust, high-performance backend services.

![A preview of the English-to-API compilation process, showing a terminal with Go code and API testing output.](/images/target-api-go.png)

### 1.1. Core Problem

Building a backend API, even a simple one, involves a significant amount of boilerplate code. Developers must define data structures, write HTTP handlers for each CRUD (Create, Read, Update, Delete) operation, set up routing, configure a server, and connect to a database. This repetitive work slows down development and is prone to human error.

### 1.2. Proposed Solution

We propose a compiler that uses English as the source language for defining API specifications. The system follows the standard three-stage pipeline to transform a high-level description into a runnable Go application:

1.  **API Description Parsing (LLM)**: An LLM parses the English input to identify API resources, their fields, and the desired endpoints.
2.  **API IR Generation**: The parsed description is converted into a structured JSON Intermediate Representation that formally defines the API's models and routes.
3.  **Go Code Generation**: A code generator processes the IR to produce a complete Go project, including model structs, HTTP handlers using the standard library or a framework like Gin, and a main server file.

## 2. API Definition Architecture

The system is designed to understand descriptions of API resources and their associated endpoints.

### 2.1. Defining Resources and Endpoints

Users describe their API in terms of resources, fields, and the HTTP methods they should respond to.

**Example Input**: `"Create a User API. The User model should have a name (string) and an email (string). Generate full CRUD endpoints for /users."`

### 2.2. API Intermediate Representation (IR)

The LLM parses this description into a structured IR that defines the API's contract.

```json
{
  "models": [
    {
      "name": "User",
      "fields": [
        { "name": "name", "type": "string" },
        { "name": "email", "type": "string" }
      ]
    }
  ],
  "routes": [
    { "path": "/users", "method": "GET", "handler": "GetUsers" },
    { "path": "/users", "method": "POST", "handler": "CreateUser" },
    { "path": "/users/{id}", "method": "GET", "handler": "GetUser" },
    { "path": "/users/{id}", "method": "PUT", "handler": "UpdateUser" },
    { "path": "/users/{id}", "method": "DELETE", "handler": "DeleteUser" }
  ]
}
```

### 2.3. Go Code Generation

The code generator uses this IR to produce the complete Go application.

-   **Models**: For each model in the IR, a corresponding Go `struct` is created.
-   **Handlers**: For each route, a Go function is generated to handle the HTTP request and response.
-   **Server**: A `main.go` file is created to set up the router and start the HTTP server.

```go
// models/user.go
package models

type User struct {
    ID    int    `json:"id"`
    Name  string `json:"name"`
    Email string `json:"email"`
}

// handlers/user_handlers.go
package handlers

import "net/http"

func GetUsers(w http.ResponseWriter, r *http.Request) {
    // ... implementation ...
}

func CreateUser(w http.ResponseWriter, r *http.Request) {
    // ... implementation ...
}

// ... other handlers ...
```

## 3. Database Integration

Most APIs require a database for persistence. The MJX engine can generate the necessary database integration code, including SQL schemas and query logic.

**Example**: `"...and connect it to a PostgreSQL database. Generate the SQL table schema."`

The IR is augmented with database information:

```json
{
  "database": {
    "type": "postgresql",
    "connectionString": "env(DATABASE_URL)"
  },
  "models": [
    // ... as before ...
  ]
}
```

The Go code generator will then:

1.  Generate a `.sql` file with the `CREATE TABLE` statement.
2.  Use the `database/sql` package in the Go handlers to connect to the database and execute queries.
3.  Optionally, integrate with an ORM like GORM or sqlx for more complex database interactions.

This allows the English-to-API engine to generate not just the API surface, but a fully functional, data-driven backend service.
